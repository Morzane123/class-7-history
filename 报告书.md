### v1.6.2 - 视频压缩进度反馈与硬件优化

**新功能:**

1. 视频上传压缩进度反馈
   - 创建进度存储模块 progressStore.ts
   - 创建进度查询API /api/events/videos/progress
   - 前端轮询获取压缩进度
   - 实时显示压缩百分比和状态

2. 小视频并行上传
   - 小于50MB的视频并行上传
   - 大于等于50MB的视频顺序上传
   - 避免服务器过载

3. 压缩未完成时等待
   - 创建/编辑页面等待所有视频上传完成
   - 显示"视频上传中..."状态
   - 上传完成后再跳转

**优化:**

1. 压缩参数调整适应服务器硬件
   - CRF: 28 → 32（降低编码质量）
   - preset: medium → veryfast（加快编码速度）
   - 最大分辨率: 1920p → 1280p
   - 最大码率: 5M → 3M

**新增文件:**
- src/lib/progressStore.ts - 进度存储模块
- src/app/api/events/videos/progress/route.ts - 进度查询API

**修改文件:**
- src/lib/videoCompress.ts - 添加进度回调、优化参数
- src/app/api/events/videos/route.ts - 集成进度存储
- src/app/events/create/page.tsx - 添加进度显示和并行上传
- src/app/events/edit/page.tsx - 添加进度显示和并行上传
- src/app/events/create/VideoUploader.tsx - 添加进度条UI

## 技术栈

- **前端**: Next.js 14 (App Router) + TypeScript + Tailwind CSS
- **后端**: Next.js API Routes
- **数据库**: SQLite (better-sqlite3)
- **认证**: JWT + bcryptjs
- **邮件**: nodemailer (腾讯企业邮箱)
- **图片处理**: sharp
- **视频处理**: fluent-ffmpeg
- **部署**: Ubuntu + PM2 + Nginx

## 部署信息

- **项目目录**: /var/www/class-7-history
- **端口**: 3003
- **域名**: class-7-history.xuanjian.top
- **进程管理**: PM2 (class-7-history)
- **反向代理**: Nginx
- **备份**: 每天03:00自动备份到/var/beifen2

## 技术决策记录

1. 选择SQLite数据库：轻量级、无需额外配置、适合小型项目
2. 选择Next.js App Router：现代化路由系统、支持服务端组件
3. 选择JWT认证：无状态、易于扩展
4. 选择Apple风格设计：简洁、现代、用户体验好
5. 选择毛玻璃效果：增强视觉层次感
6. 选择API提供文件服务：解决Next.js生产模式下的静态文件问题
7. 选择问题验证系统：确保七班用户身份真实性
8. 选择用户审核机制：增强系统安全性，防止恶意注册
9. 选择登录验证机制：保护班史内容隐私
10. 选择sharp进行图片压缩：高性能、支持WebP格式
11. 选择fluent-ffmpeg进行视频压缩：功能强大、支持多种格式
12. 选择HTTP Range请求：实现视频流式传输，优化播放体验
13. 选择轮询方式获取进度：实现简单、兼容性好
14. 选择小视频并行上传：提高上传效率，避免服务器过载
15. 选择veryfast preset：适应服务器2核CPU硬件限制
