## 开发日志

### v1.5.0 - 用户审核与安全增强

**问题修复：**
1. 添加新板块后无法在发布新事件中选择该板块
   - 原因: 事件创建页面在初始加载时获取板块列表，   - 解决: 确保每次进入创建页面时都重新获取板块列表

2. 非登录情况下可以查看班史内容
   - 解决: 在时间线和事件详情页面添加登录验证

**新功能:**

1. 用户审核功能
   - 新用户注册后默认为待审核状态 (approved=0)
   - 管理员可在后台审核或拒绝用户
   - 新用户注册时发送邮件通知管理员
   - 用户登录时检查审核状态

2. 登录验证机制
   - 时间线页面需要登录才能查看
   - 事件详情页面需要登录才能查看
   - 未登录用户显示登录提示

3. 使用条例
   - 创建使用条例页面 (/terms)
   - 注册页面添加"注册代表同意使用条例"链接
   - 条例内容包括: 用户注册、行为规范、内容管理、隐私保护等

4. 网站公告弹窗
   - 进入网站时弹出公告
   - 内容: "网络不是法外之地，请各位谨言慎行"
   - 用户关闭后不再显示 (localStorage存储)

**数据库更新:**
- 用户表新增 approved 字段 (INTEGER DEFAULT 1)

**新增API:**
- GET /api/admin/pending - 获取待审核用户列表
- POST /api/admin/pending/[id]/approve - 审核通过用户
- POST /api/admin/pending/[id]/reject - 拒绝用户注册

**新增页面:**
- /terms - 使用条例页面

**新增组件:**
- Announcement.tsx - 网站公告弹窗组件
- ClientLayout.tsx - 客户端布局包装组件

## 技术栈

- **前端**: Next.js 14 (App Router) + TypeScript + Tailwind CSS
- **后端**: Next.js API Routes
- **数据库**: SQLite (better-sqlite3)
- **认证**: JWT + bcryptjs
- **邮件**: nodemailer (腾讯企业邮箱)
- **部署**: Ubuntu + PM2 + Nginx

## 问题记录

1. PowerShell不支持`&&`语法，需使用`;`分隔命令
2. useSearchParams需要Suspense包裹
3. API路由需要标记为动态路由
4. 侧边栏状态需要提升到LayoutClient组件管理
5. Next.js生产模式下public目录的文件在构建时被静态化，运行时上传的文件需要通过API提供
6. 新板块添加后需要在创建事件页面重新获取板块列表

## 技术决策记录

1. 选择SQLite数据库：轻量级、无需额外配置、适合小型项目
2. 选择Next.js App Router：现代化路由系统、支持服务端组件
3. 选择JWT认证：无状态、易于扩展
4. 选择Apple风格设计：简洁、现代、用户体验好
5. 选择毛玻璃效果：增强视觉层次感
6. 选择API提供文件服务：解决Next.js生产模式下的静态文件问题
7. 选择问题验证系统：确保七班用户身份真实性
8. 选择用户审核机制：增强系统安全性，防止恶意注册
9. 选择登录验证机制：保护班史内容隐私
